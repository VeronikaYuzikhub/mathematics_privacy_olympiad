03 — RSA
RSA — Rivest–Shamir–Adleman
Асиметричне шифрування · 1977 · Integer Factorization Problem
1
Вибираємо два великих простих числа p і q (сотні цифр)
2
Обчислюємо n = p × q — частина публічного ключа
3
Φ(n) = (p−1)(q−1), вибираємо e (зазвичай 65537)
4
d = e⁻¹ mod φ(n) — приватний ключ
Шифр: c = me mod n
Розшифр: m = cd mod n
КЛЮЧІ
ПУБЛІЧНИЙ
(n, e)
Відкрито всім
ПРИВАТНИЙ
(d)
Тільки ви
БЕЗПЕКА
Навіть маючи c, n і e — знайти m або d практично неможливо. Задача факторизації великого числа n на p і q не вирішується за поліноміальний час.
Приклад: n = 3233 = 17 × 191, e = 17, d = 2753

03 — RSA · Демо
RSA: код та результат
RSADemo.cs
using System; using System.Numerics; using System.Text; class RSADemo { static void Main() { BigInteger n = 3233, e = 17, d = 2753; string text = "математика"; byte[] bytes = Encoding.UTF8.GetBytes(text); // Шифрування: c = m^e mod n var enc = new BigInteger[bytes.Length]; for (int i = 0; i < bytes.Length; i++) enc[i] = BigInteger.ModPow(bytes[i], e, n); // Розшифрування: m = c^d mod n var dec = new byte[bytes.Length]; for (int i = 0; i < enc.Length; i++) dec[i] = (byte)BigInteger.ModPow(enc[i], d, n); Console.WriteLine("Дешифровано: " + Encoding.UTF8.GetString(dec)); } }
ВИВІД ПРОГРАМИ
Оригінал: математика
Шифротекст: 3065 3025 3065 2816 909 3017 ...
Дешифровано: математика ✓
ЯК ПРАЦЮЄ
→
Кожен байт тексту шифрується окремо через ModPow
→
BigInteger.ModPow — ефективне модульне піднесення до степеня
→
Той самий текст відновлюється з приватним ключем d
⚠ Навчальний приклад
n = 3233 мале — для демонстрації. У реальності n має 2048–4096 біт.

04 — Diffie–Hellman
Diffie–Hellman: обмін ключами
Відкритий канал · 1976 · Discrete Logarithm Problem
ПУБЛІЧНІ ПАРАМЕТРИ
p — велике просте число
g — генератор (2, 3, 5)
КЛІЄНТ1
A = ga mod p
КЛІЄНТ2
B = gb mod p
СПІЛЬНИЙ СЕКРЕТ
Клієнт1: Ba mod p = ga·b mod p
Клієнт2: Ab mod p = ga·b mod p
Клієнт1приватний: a
Клієнт2приватний: b
↓ обмін публічними A, B ↓
shared= Ba mod p
shared= Ab mod p
g^(a·b) mod p — однаковий результат ✓
ЧОМУ ЄВА НЕ МОЖЕ ЗЛАМАТИ?
Єва бачить A, B, p, g — але щоб знайти shared, треба знати a або b. Це задача дискретного логарифму — практично нерозв'язна при великих p.

DH: код та результат
DiffieHellmanDemo.cs
using System; using System.Numerics; class DiffieHellmanDemo { static void Main() { BigInteger p = 3233; BigInteger g = 17; // Приватні ключі (генеруються локально) BigInteger a = 1234; // Клієнт1 BigInteger b = 5678; // Клієнт2 // Публічні значення BigInteger A = BigInteger.ModPow(g, a, p); BigInteger B = BigInteger.ModPow(g, b, p); // Обчислення спільного секрету BigInteger sA = BigInteger.ModPow(B, a, p); BigInteger sB = BigInteger.ModPow(A, b, p); Console.WriteLine("Спільний: " + sA); Console.WriteLine("Однакові? " + (sA == sB)); } }
ВИВІД ПРОГРАМИ
Клієнт1 надсилає: 601
Клієнт2 надсилає: 3112
Спільний ключ Клієнта1: 2816
Спільний ключ Клієнта2: 2816
Однакові? True ✓
КЛЮЧОВИЙ МОМЕНТ
Обидві сторони обчислюють однаковий спільний ключ 2816, не передаючи його по мережі. Зловмисник бачить лише 601 та 3112 — знайти a або b з цього практично неможливо.
Реальне використання
У сучасних месенджерах використовується ECDH — еліптична крива замість простих чисел, для більшої ефективності.

04.5 — Signal Protocol
Гібридний протокол: Signal
Signal Protocol поєднує кілька криптографічних примітивів для максимальної безпеки. Ключова інновація — Double Ratchet Algorithm, що оновлює ключі для кожного повідомлення.
①
X3DH — розширений обмін Diffie–Hellman для початкового встановлення сесії
②
Double Ratchet — після кожного повідомлення ключ оновлюється
③
AES-256-GCM — симетричне шифрування повідомлень
④
HMAC-SHA256 — перевірка автентичності
DOUBLE RATCHET FLOW
Початкова сесіяX3DH → Root Key
↓
Повідомлення 1Chain Key → Message Key
↓
Повідомлення 2Новий Chain Key → New Message Key
↓
…nКожне повідомлення — унікальний ключ
Forward Secrecy
Навіть якщо зловмисник отримає поточний ключ — минулі повідомлення залишаться зашифрованими.

05 — ML-KEM / Kyber
Постквантова ера: ML-KEM
NIST FIPS 203 · 2024 · Module Learning With Errors
ПРОБЛЕМА КЛАСИКИ
RSA і DH ламаються алгоритмом Шора на квантових комп'ютерах за поліноміальний час.
ML-KEM базується на LWE-задачі (Learning With Errors) — знайти s з рівняння з шумом практично неможливо навіть квантовому комп'ютеру.
1
A — випадкова матриця n×n (публічна)
2
s — секретний вектор (малі значення)
3
e — вектор помилки (-1, 0, +1) — ключовий шум
4
Публічний: b = (A·s + e) mod q
СХЕМА KYBER
Матриця An×n mod q (публічна)
↓
Секрет sмалий вектор
⊗ A·s
↓ + e (шум)
b = A·s + e mod qпублічний ключ
ПАРАМЕТРИ KYBER
3329
Модуль q
256
Розмір n
128

05 — ML-KEM · Демо
Kyber: код та результат
MLKEMTinyDemo.cs
using System; class MLKEMTinyDemo { static void Main() { int q = 97, n = 2; // Публічна матриця A int[,] A = { {51, 92}, {14, 71} }; // Секретний вектор int[] s = {60, 20}; // Вектор помилки (малий шум) int[] e = {1, 0}; // b = (A * s + e) % q int[] b = new int[n]; for (int i = 0; i < n; i++) { long sum = 0; for (int j = 0; j < n; j++) sum += (long)A[i, j] * s[j]; sum += e[i]; b[i] = (int)((sum % q + q) % q); } Console.WriteLine($"Публічний b: [{b[0]} {b[1]}]"); } }
ВИВІД (n=2, q=97)
Матриця A:
[51  92]
[14  71]
Секрет s: [60  20]
Помилка e: [1  0]
Публічний b: [51  29]
ЩО ВІДБУВАЄТЬСЯ
→
A @ s — матрично-векторне множення
→
+ e — додаємо «шум», без якого все легко зламати
→
% q — залишок по модулю q
⚠ Навчальний приклад: n=2, q=97. Реальний Kyber-512: n=256, q=3329

VulcanCore
Концептуальне ядро месенджера майбутнього
Подібно до того, як Vulkan API оптимізує векторні обчислення на GPU, VulcanCore оптимізує математичні операції криптографії — поєднуючи класику та пост-квантове в єдиному ядрі.
Векторні/матричні операції
Оптимізовані SIMD обчислення для ML-KEM та ECDH
Гібридний обмін ключами
ECDH + ML-KEM одночасно — захист зараз і в пост-квантову еру
Double Ratchet 2.0
Signal Protocol з пост-квантовим рапчетом ключів
СТЕК ТЕХНОЛОГІЙ
RSA · ECDH — класична основа
↓
Signal Protocol — гібрид + ratcheting
↓
ML-KEM / Kyber — пост-квантове
↓
VulcanCore — оптимізоване ядро
Аналогія
Vulkan API дав розробникам прямий доступ до GPU. VulcanCore дає пряме управління криптографічними примітивами без накладних витрат.

Висновки
Підсумки ·
Класичні примітиви — основа. RSA та Diffie–Hellman забезпечили безпечне спілкування на 50 років. Їхня математика — факторизація та дискретний логарифм — досі стійка до класичних атак.
Signal Protocol — гібридний стандарт. Поєднання ECDH + Double Ratchet + AES-256 забезпечує Forward Secrecy та захист навіть при компрометації одного ключа.
Постквантова ера: ML-KEM (Kyber, NIST FIPS 203, 2024) стійкий до алгоритму Шора. Вже розгортається в Chrome, Cloudflare, Signal і Apple iMessage у 2025–2026 роках.
VulcanCore — майбутнє. Оптимізовані матрично-векторні операції, аналогічно Vulkan API для GPU, дозволяють будувати ефективні та стійкі криптографічні ядра для месенджерів нового покоління.
